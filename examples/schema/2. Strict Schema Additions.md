# Strict Schema Additions

Use these examples after your starter schema is stable.

## Conditional field

```yaml
optional_fields:
  - name: project
    type: string
    severity: warning
    lint_rules:
      - rule: required_when
        field: type
        equals: [project]
        severity: error
```

## Forbidden field

```yaml
optional_fields:
  - name: archived_reason
    type: string
    severity: info
    lint_rules:
      - rule: forbidden_when
        field: status
        equals: [active]
        severity: warning
```

## Tag consistency

```yaml
required_fields:
  - name: type
    type: enum
    values: [reference, procedure, project]
    severity: error
    lint_rules:
      - rule: tag_consistency
        tag_namespace: type
        severity: warning
```

A note with `type: project` should then have a tag such as `type/project`.
